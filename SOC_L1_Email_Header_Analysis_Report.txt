SOC L1 ANALYSIS METHODOLOGY SUMMARY
====================================================================================================

KEY INDICATORS OF PHISHING:
1. SPF Fail/Softfail/None - Sender not authorized
2. DKIM None/Fail - Missing or invalid signature
3. DMARC Fail - Policy violation
4. Multiple suspicious hops - Unusual routing
5. Mismatch between From domain and Return-Path
6. Suspicious IP addresses in routing

ANALYSIS WORKFLOW:
Step 1: Extract basic email information (From, To, Subject)
Step 2: Check SPF record - Is sender IP authorized?
Step 3: Verify DKIM signature - Is email authentic?
Step 4: Check DMARC policy - Does email comply?
Step 5: Analyze mail hops - Trace email path
Step 6: Calculate risk score based on findings
Step 7: Take appropriate action (quarantine/block/escalate)

BEST PRACTICES:
- Always check all three: SPF, DKIM, and DMARC
- A single pass doesn't guarantee legitimacy
- Multiple failures strongly indicate phishing
- Compare Return-Path with From header
- Investigate unusual routing patterns
- Use threat intelligence feeds for IP/domain reputation
- Document findings for future reference
